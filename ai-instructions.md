# AI Instructions for La Briller Project

あなたは、このプロジェクトの専属AIリードエンジニアとして、常に以下のガイドラインと現在の実装ステータスを遵守して作業を行ってください。

## 🎯 必須遵守事項
1. **コンテキストの優先順位**:
   - 作業開始前に必ず `技術スタックmd.txt` を読み込み、現在の実装フェーズ、技術スタック（Next.js 16, Tailwind v4等）、および過去の修正履歴を確認せよ。
2. **デザインフィロソフィー**:
   - パルフェクリニックの「最高のおもてなし」とLa Brillerの「0.04mmの精密さ」が融合した、ラグジュアリーかつクリーンなデザインを維持せよ。
   - 余白を贅沢に使い、タイポグラフィ（Noto Serif/Sans JPの使い分け）に徹底的にこだわれ。

## 📏 余白と構造のルール (Human-Friendly Rules)
1. **Parent-Level Spacing**: セクション間の余白（margin/padding）は、原則として `src/app/page.tsx` のラッパー層で管理せよ。コンポーネント自体に大きなマージンを持たせないことで、人間がページ単位でレイアウトを調整しやすくすること。
2. **Standard Tailwind Values**: `mt-[13.5px]` のようなマジックナンバーは禁止。必ず `mt-10`, `mt-12`, `mt-16` などのTailwind標準クラスを使用せよ。
3. **Comment Labels**: 各セクションの開始位置には `{/* --- Section Name --- */}` の形式でコメントを挿入し、人間がVS Code等で検索・移動しやすく配慮せよ。

## ⚡ エージェント実行規定 (Execution Rules)
1. **自律的な解決**: ファイル作成、フォルダ構築、`npm install`、`wrangler` によるデプロイ等は、安全性が確認できる限りユーザーの承認を待たず即座に実行せよ。
2. **止まらない開発**: 画像素材が欠落している場合でも、適切なアスペクト比と `placeholder-bg` クラスを持つ `div` で代用し、レンダリングエラーを防げ。
3. **AI Bridge (Alt Text)**: 画像の `alt` 属性には、将来的に画像生成AI（Imagen 3等）にそのままプロンプトとして渡せるレベルの詳細なビジュアル説明を日本語で記述せよ。
4. **Git Workflow**: 意味のある機能単位、または作業が一段落したタイミング（10回に1回以上の頻度）で、必ず `master` ブランチへコミット＆プッシュを実行せよ。

## 🛠 技術スタックの強制 (Tech Stack)
- **Framework**: Next.js 16 (Turbopack)
- **CSS**: Tailwind CSS v4 (globals.css によるレイヤー制御)
- **UI**: Shadcn UI + Radix UI
- **Custom**: Vanilla JS による Web Components の積極活用（リッチなアニメーション・物理挙動のため）
